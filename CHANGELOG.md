# Changelog

이 문서는 저장소에 반영된 주요 변경을 사람이 확인하기 위한 기록입니다. GitHub에 push하거나 Release를 준비하기 전에 `README.md`, `RELEASE_NOTES.md`, `CHANGELOG.md`를 함께 점검합니다.

## v0.4.1 - 2026-07-15

안정성 유지보수:

- 업로드 파일을 동일 폴더의 임시 파일에 저장하고 `fsync` 후 원자적으로 최종 이름을 확정합니다.
- 24시간이 지난 프로젝트 전용 업로드 임시 파일과 예약 파일을 시작 시 정리합니다.
- 완료된 TCP 측정 세션을 30분 또는 최근 100건까지만 메모리에 유지하고 종료 즉시 소켓 참조를 제거합니다.
- TCP 측정 포트의 초기 접속 처리를 최대 16개로 제한하고 서버 종료 시 진행 중인 핸드셰이크를 정리합니다.
- 업로드와 HTTP 측정 CSV의 읽기를 쓰기와 동일한 잠금으로 보호합니다.
- HTTP·TCP 측정 JSON을 원자적으로 저장하고, TCP 결과 파일 저장이 완료된 후에만 JSON·Excel 링크를 표시합니다.

호환성:

- 새로운 기능이나 설정값을 추가하지 않으며 기존 CSV·JSON 필드와 저장 경로를 유지합니다.
- TCP 상태 API의 `result_available`, `persistence_complete` 필드는 기존 필드를 변경하지 않는 추가 정보입니다.
- `v0.4.0`과 기존 사전 릴리즈를 삭제하거나 덮어쓰지 않습니다.

## v0.4.0 - 2026-07-15

정식 릴리즈:

- `v0.4.0-rc.1`부터 `v0.4.0-rc.8`까지 검증한 HTTP·TCP 네트워크 측정 기능을 정식 배포합니다.
- 브라우저 HTTP 전송 측정과 Windows TCP 전송 성능 측정을 한 실행파일에서 제공합니다.
- 현재 서버 주소가 포함된 Windows 클라이언트 ZIP을 웹에서 받을 수 있습니다.
- 웹 결과는 실제 수신 속도, RTT와 재전송 비율 중심으로 표시합니다.
- HTTP·TCP 측정 결과를 관리자용 Excel 파일로 받을 수 있습니다.
- 파일 업로드, 메모, 저장 폴더, 직접 다운로드 링크와 CSV 기록 기능을 유지합니다.

호환성:

- `v0.4.0-rc.8`과 측정 엔진, API, JSON·CSV 형식 및 저장 경로가 같습니다.
- UDP 정밀 측정과 Android 네이티브 TCP 클라이언트는 포함하지 않습니다.
- 기존 사전 릴리즈를 보존합니다.

## v0.4.0-rc.8 - 2026-07-15

TCP 결과 표현과 관리자용 Excel:

- 사용자 화면의 이름을 `TCP 전송 성능 측정`으로 바꿨습니다.
- 업로드와 다운로드에 데이터가 이동하는 경로를 표시하고 실제 수신 평균을 대표 속도로 사용합니다.
- 방향별 결과에서 MB/s, 중앙값, 최소와 최대 속도를 함께 확인할 수 있습니다.
- RTT는 소수 둘째 자리, 재전송 비율과 방향 차이는 소수 셋째 자리까지 표시합니다.
- 재전송량은 전체 송신량 대비 비율과 함께 표시하고 CWND는 웹에서 제외해 Excel 기술 상세에서만 확인합니다.
- Excel 첫 시트를 실제 수신 속도, RTT와 재전송 중심의 관리자 요약으로 다시 구성했습니다.
- 구간별 속도에서 송신 측 기록과 수신 측 실제 값을 구분하고 스트림별 원시 통계는 상세 시트에 유지했습니다.
- 측정하지 않은 방향은 `측정 안 함`, 미지원 상세 통계는 `운영체제에서 제공하지 않음`으로 구분합니다.

호환성:

- 측정 엔진, API, JSON·CSV 형식과 저장 위치는 변경하지 않았습니다.
- 결과에 기준 속도가 없으므로 정상·비정상을 자동 판정하지 않습니다.
- 기존 `v0.4.0-rc.7` 릴리즈를 보존합니다.

## v0.4.0-rc.7 - 2026-07-14

HTTP 측정 통합과 TCP Excel:

- HTTP 측정을 하나의 탭으로 합치고 데이터량 또는 측정 시간을 종료 기준으로 선택하게 했습니다.
- HTTP 시간 기준 4개 연결을 제거하고 브라우저와 서버 모두 1개 연결만 허용합니다.
- 데이터량 기준 결과에 전송량, 걸린 시간, 최종 평균, 초당 MB와 현재 속도 기준 1GB 예상 시간을 표시합니다.
- TCP 결과 화면에서 JSON 대신 Excel을 바로 받을 수 있습니다.
- TCP Excel에 측정 요약, 송신·수신 1초 처리량 그래프와 스트림별 Windows TCP 통계를 기록합니다.
- TCP 원본 JSON과 기존 JSON 다운로드 API는 보존하며 Excel은 요청 시 메모리에서 생성합니다.
- 저장 실패 시 임시 파일과 CSV를 복구하고 측정 슬롯을 해제하는 안정성 검증을 보강했습니다.

호환성:

- 과거 HTTP 4개 연결 결과는 계속 읽을 수 있지만 새 측정에는 사용할 수 없습니다.
- UDP 정밀 측정은 추가하지 않았습니다.
- 기존 `v0.4.0-rc.6` 릴리즈를 보존합니다.

## v0.4.0-rc.6 - 2026-07-14

측정 용어와 TCP 기본 사용성:

- HTTP 측정 탭을 `HTTP 용량 기준`과 `HTTP 시간 기준`으로 구분해 각각 데이터량과 시간으로 종료된다는 점을 명확히 했습니다.
- TCP 정밀 측정을 기본 활성화하고, 이전 기본값 `false`는 업데이트 후 한 번만 `true`로 전환합니다.
- 설정 마이그레이션을 마친 뒤 사용자가 TCP 측정을 다시 끄면 그 선택을 유지합니다.
- TCP 5201 포트가 사용 중이면 다음 빈 포트를 제안하고, 승인과 실제 바인딩을 모두 통과한 포트만 설정에 저장합니다.
- TCP 포트를 준비하지 못해도 파일 업로드와 HTTP 측정 서버는 계속 실행합니다.
- TCP 탭 상단에 자동 연결 주소가 포함된 Windows 클라이언트 ZIP 다운로드를 주 작업으로 배치했습니다.
- TCP 데이터 포트는 클라이언트 등록 후 자동 전달되므로 서버 IP 또는 웹 포트가 바뀐 경우에만 ZIP을 다시 받습니다.

호환성:

- 측정 알고리즘과 API, CSV·JSON 결과 형식은 유지합니다.
- Windows 방화벽 규칙은 자동 변경하지 않고 현재 TCP 포트의 수동 허용 명령을 안내합니다.
- 기존 `v0.4.0-rc.5` 릴리즈를 보존합니다.

## v0.4.0-rc.5 - 2026-07-14

HTTP 지속 측정 진행률:

- 단계마다 0%로 돌아가던 진행바를 전체 측정 기준의 단일 진행률로 변경했습니다.
- 응답시간 확인은 5%, 워밍업과 본 측정은 실제 시간 비율에 따라 나머지 95%를 사용합니다.
- 업로드 상태 조회와 다운로드 청크 수신에서 진행바를 분리해 브라우저 시간 기준으로 부드럽게 움직입니다.
- 현재 단계, 전체 퍼센트와 해당 단계 경과 시간을 한 문구에서 확인할 수 있습니다.
- 성공한 경우에만 100%로 표시하며 실패·취소 시에는 중단 위치를 유지합니다.

호환성:

- 전송 방식, 측정값, 1초 그래프, 결과 저장 형식과 서버 API는 변경하지 않았습니다.
- 기존 `v0.4.0-rc.4` 릴리즈는 보존합니다.

## v0.4.0-rc.4 - 2026-07-14

HTTP 지속 측정 Excel 보고서:

- 측정 화면에서 JSON 대신 Excel 결과를 바로 받을 수 있습니다.
- `측정 요약` 시트에 상태·오류·세션·측정 조건·HTTP 응답시간과 업로드/다운로드 통계를 기록합니다.
- `구간별 속도` 시트에 방향별 1초 Byte·Mbps와 선 그래프를 기록합니다.
- 평균 처리량은 Mbps와 MB/s, 전송량은 Byte와 MiB로 함께 표시합니다.
- 성공·실패·취소 결과를 모두 지원하고 실패 시에도 수집된 구간은 보존합니다.
- Excel은 세션 JSON에서 요청 시 메모리로 생성하며 서버에 별도 파일을 남기지 않습니다.

호환성과 안전성:

- 기존 JSON 저장과 다운로드 API는 호환성을 위해 유지하되 화면 버튼에서는 노출하지 않습니다.
- 기존 측정 PC IP 제한과 다운로드 캐시 방지를 Excel에도 적용합니다.
- Excel 셀의 외부 입력은 수식으로 실행되지 않도록 처리합니다.
- 별도 측정 위치·AP·메모 입력과 정상/경고 판정은 추가하지 않습니다.

- `BASE_URL`이 기존 웹 포트를 사용할 때만 새 포트로 함께 변경합니다.
- 변경된 포트의 Windows 방화벽 상태를 확인하고 필요한 수동 허용 명령을 안내합니다.

- 시작 CMD와 안내문이 고정된 8000번 대신 콘솔에 표시되는 실제 접속 주소를 사용하도록 안내합니다.
- 새 포트의 웹 화면에서 생성한 TCP 클라이언트 ZIP에는 해당 포트가 자동으로 포함됩니다.
- TCP 측정 포트 `5201` 자동 변경과 방화벽 자동 수정은 포함하지 않습니다.
- 기존 `v0.4.0-rc.2` 릴리즈는 보존합니다.

## v0.4.0-rc.2 - 2026-07-10

TCP 클라이언트 배포 개선:

- TCP 정밀 측정 웹 화면에서 Windows 클라이언트 ZIP을 직접 받을 수 있습니다.
- 현재 브라우저가 접속한 서버 PC 이름 또는 IPv4와 웹 포트를 자동 연결 CMD에 포함합니다.
- localhost 접속은 자동 감지한 사내 IPv4로 대체하고, 안전하지 않은 Host 값과 IPv6는 차단합니다.
- 클라이언트는 ZIP을 완전히 해제한 뒤 CMD만 실행하면 자동 등록되어 측정 작업을 기다립니다.
- 서버 IP 또는 웹 포트가 변경되면 새 ZIP을 다시 받는 단순한 운영 방식을 사용합니다.
- 클라이언트 ZIP에는 `config.ini`, 인증 토큰, 세션 데이터와 운영 기록을 포함하지 않습니다.

검증과 호환성:

- EXE 자체 점검에 클라이언트 ZIP 생성·구조 검증을 추가했습니다.
- SHA256 파일을 LF 줄바꿈으로 생성해 Windows, macOS와 Linux에서 동일하게 검증할 수 있습니다.
- 전체 Release ZIP의 주소 입력형 CMD는 자동 ZIP을 받을 수 없을 때의 수동 대체 수단으로 유지합니다.
- UDP, 자동 서버 탐색, 자동 압축 해제·실행과 Android 네이티브 클라이언트는 포함하지 않습니다.

## v0.4.0-rc.1 - 2026-07-10

자체 TCP 정밀 측정:

- 같은 `InternalUpload.exe`를 서버와 대기형 Windows 측정 클라이언트로 실행할 수 있습니다.
- HTTP 제어 API와 별도 TCP 5201 데이터 연결을 분리했습니다.
- 업로드·다운로드·전체, 1개·4개 스트림, 3초 워밍업, 10초·30초 측정을 지원합니다.
- 송신·수신 처리량, 1초 구간 그래프와 Windows TCP_INFO 기반 RTT·CWND·재전송 바이트를 기록합니다.
- PC 이름과 접속 IP로 에이전트를 구분하고 IP에 결합된 임시 토큰으로 데이터 연결을 검증합니다.
- 간편·지속·TCP 네트워크 측정을 서버 전체에서 한 번에 하나만 허용합니다.
- TCP 요약 CSV와 세션별 상세 JSON을 별도 저장합니다.

배포와 검증:

- ZIP에 `start_tcp_probe_client.cmd`, TCP CSV 헤더와 결과 폴더 안내를 추가했습니다.
- EXE에 `--probe-self-check` loopback 검증을 추가했습니다.
- GitHub Actions가 `-rc.N` 태그를 prerelease로 만들고 ZIP과 SHA256 파일을 함께 게시합니다.
- 이 RC는 자동 loopback 검증 대상이며 Windows 두 PC iperf 중앙값 대비 ±10% 실기 검증 후 정식 `v0.4.0`으로 승격합니다.

제외 항목:

- iperf 실행파일·라이브러리·호환 프로토콜, UDP, Android 네이티브 에이전트는 포함하지 않습니다.

## v0.3.0 - 2026-07-10

HTTP 지속 네트워크 측정:

- 기존 크기 기준 간편 측정을 유지하고 3초 워밍 후 10초/30초를 재는 지속 측정을 추가했습니다.
- HTTP 연결 1개/4개와 업로드, 다운로드, 전체 측정을 지원합니다.
- 평균·중앙·최소·최대·변동률, HTTP 응답시간과 1초 구간 그래프를 표시합니다.
- 동시 세션을 1건으로 제한하고 취소·만료 상태도 기록합니다.
- 지속 측정 요약 CSV와 세션별 상세 JSON을 분리해 저장합니다.
- 요청 스트리밍을 사용하지 않아 Edge/Chrome + HTTP/1.1 환경의 기존 안정성을 유지합니다.
- Windows 11 Edge/Chrome과 Android Chrome 화면을 검증했습니다.

안정성 개선:

- 업로드 파일 저장 후 CSV 기록이 실패하면 새로 생긴 고아 파일을 정리합니다.

제한사항:

- HTTP 지속 측정은 iperf와 같은 TCP·UDP 정밀 측정이 아닙니다.
- `v0.4.0` TCP와 `v0.5.0` UDP 측정은 후속 계획이며 현재 릴리즈에 포함되지 않습니다.

## v0.2.2 - 2026-07-09

네트워크 체크 표시 개선:

- 진행 중 속도를 평균 속도와 구간 속도로 나눠 표시합니다.

- 업로드, 다운로드, 삭제, 경로 검증, 링크 생성, CSV 기록에 대한 pytest 테스트를 추가했습니다.
